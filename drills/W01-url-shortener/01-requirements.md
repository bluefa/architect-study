# W01 URL Shortener - Requirements (샘플)

> 형식 예시용 최소 샘플입니다. 수치는 학습자가 직접 보정해야 합니다.

## 1) 문제 정의
- 한 줄 정의: 긴 URL을 짧은 링크로 변환하고 빠르게 리다이렉트한다.
- 사용자/고객: 링크를 공유하는 일반 사용자, 캠페인 운영자
- 성공 기준: 생성 성공률과 리다이렉트 지연시간 목표 달성

## 2) 목표 / 비목표
| 구분 | 내용 | 측정 방법 |
|---|---|---|
| Goal 1 | 짧은 URL 생성 API 제공 | 성공률 99.9% 이상 |
| Goal 2 | 리다이렉트 빠른 응답 | p95 100ms 이하 |
| Non-goal 1 | 사용자 대시보드 고도화 | W01 범위 제외 |
| Non-goal 2 | 커스텀 도메인 다중 지원 | 추후 과제 |

## 3) 트래픽 / 용량 가정
| 항목 | 현재 가정 | 12개월 가정 | 근거 |
|---|---:|---:|---|
| DAU | 50,000 | 200,000 | 학습용 가정 |
| QPS (Read) | 1,000 | 5,000 | 생성 대비 조회 다수 |
| QPS (Write) | 100 | 500 | 신규 생성량 기준 |
| 데이터 저장량 | 20GB | 200GB | URL + 메타데이터 |

## 4) SLA / SLO
| 지표 | 목표 | 측정 윈도우 | 비고 |
|---|---|---|---|
| 가용성 | 99.9% | 월간 | API+redirect |
| 지연시간 p95 | 100ms 이하 | 5분 | redirect API |
| 에러율 | 0.1% 이하 | 5분 | 5xx 기준 |

## 5) 정합성 / 데이터 보존
- 정합성 수준: 최종 정합성 허용 (생성 직후 짧은 지연 허용)
- RPO: 5분
- RTO: 30분

## 6) 제약 조건
- 기술/인력/기간: 1인, 1주
- 규제/보안: 악성 URL 차단 정책 필요
- 예산: 학습 환경 기준 저비용 우선
