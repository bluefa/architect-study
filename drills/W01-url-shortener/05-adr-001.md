# ADR-001: URL Key 발급 전략 (샘플)

## Context
- URL Shortener에서 키 충돌 없이 빠른 생성이 필요하다.
- W01 범위는 학습용이므로 운영 복잡도는 낮게 유지한다.
- 초기 트래픽은 중간 수준으로 가정한다.

## Decision
- 기본 전략으로 중앙 시퀀스 기반 키 발급을 채택한다.
- 발급기 장애 시 제한적으로 랜덤 키 fallback 경로를 둔다.

## Consequences
| 구분 | 내용 |
|---|---|
| Positive | 구현 단순, 디버깅 용이, 예측 가능한 지연시간 |
| Negative | 발급기 병목/SPOF 위험이 존재 |
| Neutral | 트래픽 증가 시 shard 전략 재평가 필요 |

## Alternatives Considered
| 대안 | 장점 | 단점 | 기각 이유 |
|---|---|---|---|
| 랜덤 키 + 중복 확인 | 분산 발급 용이 | 충돌 처리, 재시도 복잡 | W01 범위 대비 복잡도 높음 |
| 사전 생성 키 풀 | 빠른 응답 가능 | 풀 관리/고갈 처리 필요 | 운영 부담 증가 |
